# git
1. 设置用户名（替换为你的名称，如 GitHub 用户名）
git config --global user.name "Your Name"

2. 设置邮箱（替换为你的邮箱，建议与 GitHub/GitLab 绑定邮箱一致）
git config --global user.email "your.email@example.com"

3. 查看所有全局配置
git config --global --list

# git连接GitHub
1. Git生成安全密钥
	bash>>ssh-keygen -t ed25519 -C "your_email@example.com"
	一路回车
	bash>>cat ~/.ssh/id_ed25519.pub
	复制内容
	GitHub>>头像>>设置>>SHH**>>新增SHH
2. 先测试基础连接
在 Git Bash 中执行以下命令，确认是否是 GitHub 服务器或网络问题：
	bash>>ping github.com
3.改网关(开了加速可以不用设置）
	阿里公共 DNS：223.5.5.5
	腾讯 DNSPod：119.29.29.29
4.刷新网关缓存(开了加速可以不用设置）
	cmd管理员>>>ipconfig /flushdns 
5.改用 443 端口（推荐）
	在此自前要生成安全密钥
	22 端口可能被防火墙或网络服务商封锁，而 443 端口（HTTPS 常用）通常开放。可以配置 SSH 通过 443 端口连接 GitHub：
	创建 / 编辑 SSH 配置文件：
	找到文件：C:\Users\ZGN\.ssh\config，复制粘贴下面的话，格式不能错
Host github.com
    Hostname ssh.github.com
    Port 443

5. 再次测试连接
	执行测试命令，确认是否成功：
	bash>>>ssh -T git@github.com
		首次连接会提示 Are you sure you want to continue connecting?，输入 yes 回车。
		成功会显示：Hi 你的用户名! You've successfully authenticated...
6. GitHub上现在默认的是以main作为默认分支>>那就需要把git的默认分支也改成main
	# 语法：git branch -m 旧分支名 新分支名
	bash>>>git branch -m master main
